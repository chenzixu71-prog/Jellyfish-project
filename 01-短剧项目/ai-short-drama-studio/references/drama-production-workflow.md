# AI Short Drama Production Workflow

## Default Pilot Scope

Use this scope unless the user specifies otherwise:
- 8 pilot episodes
- 60-90 seconds per episode
- 9:16 vertical video
- live-action realistic image style
- 2-4 main recurring characters
- one primary location cluster to reduce asset drift

## Pipeline

1. Market and genre choice
   - Pick one genre promise: revenge, rebirth, identity reveal, romance conflict, family conflict, workplace comeback, supernatural system, or suspense.
   - Define the emotional engine: humiliation -> reversal, betrayal -> revenge, hidden identity -> reveal, forbidden love -> pressure, ordinary person -> power shift.

2. Premise
   - One sentence: "A [protagonist] who [wound/secret] must [goal] before [stakes], but [antagonist pressure]."
   - Add the first 3-second hook and the first episode cliffhanger before expanding the season.

3. Character bible
   - Create recurring roles before writing scenes.
   - Use `character-consistency.md` and `micro-expression-library.md`.

4. Season arc
   - Split into setup, escalation, reversal, exposure, final payoff.
   - Every 3-5 episodes should reveal one new fact or reverse one power relationship.

5. Episode beats
   - Each episode needs: opening conflict, pressure, information reveal, emotional turn, cliffhanger.
   - Avoid low-conflict exposition.

6. Script
   - Keep dialogue short and playable.
   - Write visible actions, not inner monologue, unless it becomes voiceover.
   - Mark voiceover, dialogue, subtitle emphasis, and sound effects.

7. Shot list
   - One emotional beat per shot.
   - 5-10 shots per 60-90 second episode is a practical starting point.
   - Prefer close-up, medium close-up, over-shoulder, hand/object insert, phone screen insert, and reaction shots.

8. Image generation
   - Generate character reference sheets first.
   - Generate scene keyframes only after identity lock is accepted.

9. Image-to-video
   - Use reference image/first frame.
   - Keep clips short.
   - Avoid complex multi-person blocking unless the tool handles it reliably.

10. Voice, subtitles, and edit
   - Build voice profiles per character.
   - Generate subtitles from final script.
   - Cut for hook density and emotional readability.

11. Platform packaging
   - Prepare title, cover, synopsis, tags, episode list, copyright notes, tool-use notes, and compliance checklist.

## Minimum Deliverable

For a complete pilot package, output:
- project one-liner
- audience and genre promise
- character bibles
- expression prompt bank
- 8-episode outline
- episode 1 full script
- episode 1 shot list
- image prompts
- image-to-video prompts
- voice/subtitle table
- QA checklist
