# 水母diy学习助手 - UI 视觉与动效规范

## 1. 视觉方向

本项目 UI 从普通教育工具风格，调整为更有国际化品牌感的手绘学习产品风格。

参考方向：

- 手绘咖啡品牌感。
- 大面积留白。
- 粗黑线条插画。
- 红、黄、蓝高饱和点缀。
- 手写字标题和轻松的 Q 版人物。
- 页面整体保持干净，不做复杂渐变和重装饰。

关键词：

```text
playful
hand-drawn
international
bold color
warm paper
learning coach
celebration
```

## 2. 色彩规范

主色建议：

| 用途 | 颜色 | 说明 |
| --- | --- | --- |
| Brand Blue | `#1767C8` | 主按钮、导航、重点图形 |
| Tomato Red | `#E5382F` | 正确反馈、重点字、彩带 |
| Sunny Yellow | `#FFE142` | 标签、奖励、提示 |
| Ink Black | `#111111` | 手绘线条和主标题 |
| Warm Paper | `#FFFDF4` | 页面背景 |
| Soft Gray | `#F2F3F5` | 卡片底和分割 |

避免：

- 全页面单一蓝色。
- 大面积深色背景。
- 过度科技感渐变。
- 过度圆润的幼儿化风格。

## 3. 字体与文字风格

中文正文：

```text
Noto Sans SC / PingFang SC / Microsoft YaHei
```

英文和品牌字：

```text
手写风格标题，可在 Figma 中用手绘 SVG 或图片资产承载
```

标题策略：

- 首页标题可以使用更大、更有手写感的字形。
- 普通页面仍使用清晰字体，保证小程序阅读体验。
- 重点词可用蓝、红、黄三色拆字处理，例如“AI 闯关”“学习报告”。

## 4. 角色资产

角色不再使用水母。

第一版使用两个 Q 版学习伙伴：

| 角色 | 用途 |
| --- | --- |
| Q 版小男孩 | 输入页、学习引导、报告鼓励 |
| Q 版小女孩 | 生成中、答题反馈、庆祝状态 |

角色画风：

- 粗黑线条。
- 简单五官。
- 大头小身体。
- 可爱但不幼稚。
- 表情至少包含：idle、smile、laugh、thinking。

## 5. 图案资产

UI 中可以适量加入以下图案，但不要抢主内容：

- 彩带。
- 星星。
- 铅笔。
- 笔记本。
- 对话气泡。
- 问号。
- 勾号。
- 奖章。
- 手绘箭头。
- 小圆点和不规则色块。

使用位置：

- 首页输入卡片周围。
- 生成中 loading 区域。
- 答题完成庆祝层。
- 报告页顶部。
- 分享海报背景。

## 6. 核心页面调整

### 6.1 输入页

目标：让用户感觉这是一个轻松的学习入口，不是表单工具。

页面元素：

- 手写风格品牌标题。
- Q 版小男孩或小女孩在输入框旁边。
- 大输入卡片。
- “生成闯关题”主按钮。
- 少量手绘图案围绕输入卡片。

### 6.2 生成中页面

目标：让等待过程有陪伴感。

页面元素：

- Q 版角色 thinking / smile 状态。
- loading 文案：

```text
AI 正在帮你整理知识点...
```

- 三个阶段提示：

```text
提取知识点
生成题目
准备讲解
```

### 6.3 答题页

目标：卡片化、轻量、适合初学者。

页面元素：

- 顶部进度：`第 1 / 5 题`。
- 题型标签：单选 / 多选 / 判断。
- 题目卡片。
- 选项按钮。
- 提交按钮。
- 答对/答错后显示即时讲解。

### 6.4 答题完成庆祝页

用户完成 5 题后，不直接进入报告。

先展示庆祝层：

```text
恭喜完成本轮闯关！
```

然后出现按钮：

```text
查看学习报告
```

## 7. 答题完成动效

### 7.1 彩带飘洒

触发时机：

```text
用户完成第 5 题并提交答案后
```

动效流程：

| 时间 | 状态 |
| --- | --- |
| 0ms | 页面中央弹出“恭喜完成本轮闯关” |
| 0-200ms | 标题从 0.92 放大到 1.04 |
| 200-450ms | 标题回弹到 1.00 |
| 0-800ms | 彩带从顶部快速飘洒 |
| 800-1200ms | 彩带透明度从 1 渐隐到 0 |
| 900ms | “查看学习报告”按钮出现 |

彩带样式：

- 红、黄、蓝三色。
- 形状包含短条、圆点、小三角。
- 不超过 24 个元素，避免卡顿。

Taro 实现建议：

```text
使用 CSS animation + 条件渲染，不需要引入重动画库。
```

### 7.2 查看报告按钮

按钮出现规则：

```text
彩带开始渐隐后出现
```

按钮文案：

```text
查看学习报告
```

点击后进入报告生成 loading。

## 8. 报告生成交互

### 8.1 点击查看报告

用户点击：

```text
查看学习报告
```

前端请求：

```text
POST /api/generate-report
```

### 8.2 Loading 状态

页面文案：

```text
loading......
AI 正在生成你的学习报告
```

视觉：

- Q 版角色 thinking 状态。
- 手绘进度点。
- 背景少量星星和问号图案。

### 8.3 报告完成

加载完成后展示完整报告：

- 本轮得分。
- 掌握度。
- 错题列表。
- 薄弱知识点。
- 下一步学习建议。
- 再来一组题。
- 分享到微信。
- 分享到 QQ。

## 9. 分享能力

第一版报告页支持两个分享入口：

```text
分享到 VX
分享到 QQ
```

小程序现实约束：

- 微信小程序原生支持微信分享。
- QQ 分享能力取决于平台能力和发布环境，第一版可以先保留 UI 入口。
- 如果 QQ 分享无法直接调用，先生成报告海报，用户保存后手动分享。

推荐实现：

| 分享方式 | P1 实现 |
| --- | --- |
| VX | 使用微信小程序分享能力 |
| QQ | 先做 UI 入口，后续根据平台能力接入 |
| 海报 | P2 再做图片生成 |

## 10. 前端状态机

报告相关状态：

```text
quiz_answering
quiz_completed
celebrating
report_waiting_click
report_loading
report_ready
report_share_panel
```

状态流：

```text
答完第 5 题
-> celebrating
-> report_waiting_click
-> 点击查看学习报告
-> report_loading
-> report_ready
-> 分享到 VX / QQ
```
