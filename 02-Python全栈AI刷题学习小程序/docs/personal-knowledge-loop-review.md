# 个人知识库学习闭环评估

## 结论

下一轮只做一条主线：**真实资料入库 -> 可补充资料和题目 -> 点击知识库直接闯关 -> 下次继续上次知识库**。排行榜、公开题库、会员和复杂社交继续后置。

## 三角色评估

### 产品

- 首页知识库卡片主区域点击后直接生成关卡并进入闯关；“管理”作为独立入口。
- 用户最后一次选择的知识库是账号级学习状态；游客保留本地副本，登录后合并。
- 资料解析失败必须显示失败原因，不能用文件名或“上传了图片”继续生成通用题。
- 手工补题第一版支持单选、多选、判断，字段与现有题目一致。

### 开发

- 后端已有素材上传建库、素材补充、知识库出题接口，可以复用。
- 前端目前只调用 JSON 文本接口，图片没有真正上传，必须改为 multipart 上传。
- `activeKnowledgeBaseId` 需要同时支持游客本地保存和登录用户后端保存。
- 知识库需增加手工题集合及新增、编辑、删除接口；组卷时优先保留手工题原文。

### AI 算法

- 当前中文长文可能被切成一个大块，且总内容会静默截断，无法保证长文档覆盖。
- OCR 失败目前会被当成普通内容继续处理，必须改成 `parsed/partial/failed` 状态。
- 每道 AI 题应保存证据块 ID；无可靠证据时应少出题或提示材料不足。
- 解析链路需要保留文件名、类型、hash、页码/位置、解析状态和失败原因。

## 需求批次

### P0-A 真实资料入库

- 格式：首批 `txt/md/csv/json/pdf/docx/jpg/jpeg/png`。
- 限制：最多 3 个文件、10 张图片；逐个显示上传和解析状态。
- 规则：真实类型校验、文件 hash 去重、中文段落分块、OCR 失败可重试。
- 验收：图片或文件内容能在后续题目中被命中；失败素材不会生成题目。

### P0-B 知识库补充

- 支持补充文字、文件、图片和手工题。
- 手工题字段：题型、题干、选项、答案、讲解、知识点、难度。
- 重复资料不重复入库；手工题重新登录后保持不变。

### P0-C 点击即闯关

- 点击知识库卡片主区域：记录当前知识库 -> 显示生成态 -> 进入闯关。
- 点击管理按钮：进入知识库详情，不触发出题。
- 同一知识库重复进入时生成新一轮题组，保留历史题组关联。

### P0-D 恢复上次知识库

- 游客：本地保存 `activeKnowledgeBaseId`。
- 登录用户：后端保存账号级 `activeKnowledgeBaseId`，登录后以后端为准。
- 首页展示“继续上次闯关”；知识库已删除时清理状态并回到列表。
- 本批只恢复所选知识库；恢复未完成题号和答案作为 P1。

### P0-E 登录边界

- 登录入口只保留微信一键登录、手机号验证码登录。
- 知识库归属统一使用 `userId`，微信和手机号只是登录凭证，避免同一用户数据分裂。
- 游客登录后迁移知识库和当前选择；超过 5 个时必须返回冲突清单，不能静默丢弃。

### P0-F 平台安全与一致性

- 请求带登录 token 时必须验证成功；失效 token 返回重新登录，不能静默降级为游客。
- 出题响应不返回正确答案和完整讲解，用户提交后才返回判定依据。
- 知识库补充和生成闯关增加版本检查，避免并发操作覆盖资料或 `quizIds`。
- 游客标识不能只信任任意客户端 `sessionId`，需要服务端签名或等价保护。

## 明确后置

- P1：未完成题次恢复、素材删除/重解析、手工题编辑和组卷比例。
- P2：向量数据库、公开题库、协作、排行榜、VIP、视频和音频解析。

## 验收总线

1. 上传真实文件/图片后可看到解析状态、来源和有效内容摘要。
2. 补充资料或手工题后，下一轮闯关能命中新增内容。
3. 首页点击知识库一次即可进入对应闯关。
4. 退出并重新登录后仍能找到上次选择的知识库。
5. 微信与手机号登录共享统一用户数据，游客迁移不重复。
6. 后端单元/API 测试、Taro 微信构建和微信开发者工具人工链路全部通过。

## 决策门

建议先确认上述 P0-A 至 P0-F，再进入 TDD 开发。当前实现不能直接视为“上传图片已建库”，因为前端尚未上传图片二进制内容。
