# 题库与闯关扩展需求

状态：已确认，2026-07-16。

## 核心规则

- 每个用户最多创建 5 个知识库。
- 每个知识库最多累计 200 道有效去重题目；200 是容量上限，不是单轮题量。
- 首次生成 20 题，每轮闯关 10 题；未学习库存不足 10 题时再生成 20 题。
- 知识库进度为“已首次提交的去重题数 / 当前有效题目总数”；重复练习不增加进度。
- 题型包括单选、多选、判断和问答。

## 问答题

流程：用户回答 -> 查看参考答案和知识点 -> 用户选择“我答对了”或“还没掌握” -> 保存自评 -> 下一题。

- 不会作答时可以直接看答案，默认记为“还没掌握”。
- 问答题结果标记为用户自评；报告与客观题正确率分开统计。
- “还没掌握”的问答题进入错题本。

## 页面链路

首页只负责创建知识库和补充学习来源。闯关 Tab 展示知识库列表；用户点击知识库后进入详情，再开始或继续闯关。

- 知识库列表显示累计进度，例如 `37/126`。
- 答题页同时显示知识库累计进度和本轮进度，例如 `37/126`、`3/10`。
- 新链路覆盖旧的“点击知识库立即生成一轮题”规则。

## 搜索边界

- 关闭联网搜索：Tavily Search 和 Extract 调用次数必须为 0，只解析用户文字、文件和图片；DeepSeek 只能基于这些材料出题。
- 关闭联网搜索且只输入 URL：提示用户开启联网获取网页内容。
- 开启联网搜索：关键词可 Search，URL 可 Extract；失败时保留本地材料并明确提示降级。
- 材料不足时允许少出题，不得使用通用题凑数。

## 开发批次

1. P0：题目池、200 题上限、来源与累计进度模型。
2. P1：闯关 Tab 知识库列表及开始/继续链路。
3. P2：首次 20、每轮 10、库存不足自动补 20。
4. P3：问答题、自评、错题和报告拆分统计。
5. P4：搜索零调用保护、降级提示及完整测试。
