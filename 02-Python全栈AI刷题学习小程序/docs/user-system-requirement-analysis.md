# 水母diy学习助手 - 用户系统需求分析

## 0. 文档状态

本文档只做用户系统需求分析与批次排序，等待人工确认后再进入方案设计和开发。

## 1. 一句话判断

用户系统第一阶段的核心不是“做完整账号体系”，而是让用户能登录、能保存学习数据、能回看自己的闯关记录。

## 2. 用户与场景

主要用户：

- 已经完成一次或多次闯关，希望后续继续复习的人。
- 想查看历史报告、错题、学习进度的人。
- 换设备后仍希望找回学习记录的人。

核心场景：

```text
用户完成学习 -> 登录保存数据 -> 再次进入小程序 -> 查看个人信息和历史学习记录
```

## 3. 需求批次与优先级

### P0 必做：用户登录

目标：用户可以用微信完成最基础登录。

范围：

- 微信一键登录。
- 登录成功后保存登录态。
- 未登录时仍可继续游客模式学习。
- 登录失败时给出明确提示。

不做：

- 手机号登录。
- 密码注册。
- 复杂个人资料编辑。

验收：

- 用户点击登录后能进入已登录状态。
- 退出小程序再进入，短期内仍保持登录。
- 不登录也不影响原有学习链路。

### P1 必做：获取用户信息

目标：登录后能展示用户基础资料和学习概览。

范围：

- 展示默认昵称或微信昵称。
- 展示默认头像或用户头像。
- 展示累计答题数、正确率、学习次数。
- 展示等级、经验值、连续学习天数。

验收：

- “我的”页面能区分未登录和已登录状态。
- 已登录用户能看到自己的基础学习数据。

### P2 必做：游客数据绑定

目标：用户登录后，登录前产生的学习数据不能丢。

范围：

- 当前游客 `sessionId` 下的闯关、报告、错题绑定到登录用户。
- 重复登录时不重复导入同一批数据。

验收：

- 用户先游客答题，再登录，历史记录仍可查看。
- 用户再次登录，不出现重复报告或重复错题。

### P3 必做：闯关历史列表

目标：用户可以回看自己完成过的学习任务。

范围：

- 展示历史闯关列表。
- 每条记录包含学习主题、完成时间、得分、掌握度。
- 点击记录可进入对应学习报告。

验收：

- 用户完成多次闯关后，“我的/记录”中能看到列表。
- 点击历史记录能查看报告详情。

### P4 应做：报告历史

目标：用户可以集中查看历史学习报告。

范围：

- 展示最近报告列表。
- 支持查看报告详情。
- 保留分享入口。

验收：

- 报告生成后能被保存。
- 用户之后能从报告历史重新打开。

### P5 应做：错题本

目标：用户可以复习曾经答错的题。

范围：

- 展示错题列表。
- 展示题干、用户答案、正确答案、知识点讲解。
- 支持按薄弱点生成一组复习题。

验收：

- 答错的题能进入错题本。
- 用户能从错题本查看讲解。

### P6 可后置：成长体系

目标：提升持续学习动力。

范围：

- 等级。
- 经验值。
- 连续学习天数。
- 基础勋章。

验收：

- 完成学习后经验值增加。
- 达到条件后展示勋章。

## 4. 本期不做

- VIP、充值、额度付费。
- 好友、排行榜、PK。
- 手机号登录。
- 密码账号体系。
- 管理后台。
- 多设备实时同步冲突处理。
- 用户主动编辑复杂资料。

## 5. 推荐实现顺序

```text
1. 用户登录
2. 获取用户信息
3. 游客数据绑定
4. 闯关历史列表
5. 报告历史
6. 错题本
7. 成长体系
```
