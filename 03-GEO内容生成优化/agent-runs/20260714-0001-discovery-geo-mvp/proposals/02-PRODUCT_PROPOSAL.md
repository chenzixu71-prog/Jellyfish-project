# Agent 2 独立提案：AI Visibility Sprint Workspace

## Target User And Job

- 用户：负责 B2B/SaaS 获客内容的市场或增长负责人。
- 任务：用一个项目工作区完成品牌基线、竞品差距、内容修复和结果汇报。

## Evidence

- Peec、Semrush、Profound 等产品采用项目/提示词/来源/模型/报告结构，说明工作区模式易于承载任务。
- 当前简历目标需要同时展示产品、前后端、数据和迭代能力。

## Problem And Value

用户面对的不是单次写作问题，而是一项需要配置、评估、采取行动和汇报的增长任务。产品应体现项目状态和闭环，而不只是一个文本框。

## Proposed MVP

1. 创建品牌项目并加载 Demo。
2. 生成并管理问题集。
3. 运行 AI 可见性基线和竞品对比。
4. 查看 GEO 规则诊断和优先级。
5. 生成内容优化版本并记录采纳行为。
6. 重跑评估并保存报告。

## Success Metrics

- 项目创建到基线报告的激活率。
- 核心闭环完成率。
- 优化建议采纳率。
- Before/After 规则与可见性变化。

## Risks And Non-goals

- 风险：工作区容易扩张为多项目、历史趋势和团队协作。
- 非目标：支付、团队权限、大规模定时任务和企业级报告系统。
